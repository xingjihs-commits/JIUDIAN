## 工作机制

### 执行流程

```
N系列EXE 放在桌面
      │
      ▼
双击/bat执行
      │
      ▼
读取自身文件名:
  "CardLock" = 固定前缀
  "N" = 标记为授权注入器
  "8.9.1" / "02_d" = 密钥参数
      │
      ▼
修改 System.ini:
  ├── LD  ← 用文件名派生的密钥产生新的32字节授权码
  └── LDO ← 设置新的到期日期
      │
      ▼
校验通过 → CardLock.exe 正常运行
校验失败 → N9101/E0073 弹窗锁死
```

### 关键特征

- **文件名即密钥**：文件名的 N 后缀（如 `N8.9.1`、`N02_d`）是生成 LD 的种子
- **自毁脚本**：执行完的 bat 脚本会自动删除，只留 Prefetch 痕迹
- **不修改其他文件**：只动 System.ini 的 LD/LDO 字段
- **可远程注入**：通过 UltraViewer 文件传输功能投递

---

## 发现过程

1. **UltraViewer 聊天记录**（`126931620.uvcl`）中厂家明文传输了 N系列 EXE 的文件名
2. **Prefetch 文件分析** 确认了 N系列 EXE 在酒店电脑上的执行痕迹
3. **System.ini 4个版本对比** 确认了 LD/LDO 在不同时间点的变化
4. **文件名反推** 发现 N 后缀的特定模式是生成 LD 的密钥种子
