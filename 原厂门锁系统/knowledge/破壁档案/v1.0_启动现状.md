## 9. 2026-05-30 门锁卡接管结论

本节记录本次关于老门锁系统卡逻辑的逆向结论与 Solid 已落地改动。结论以 `酒店系统/tools/dev/_legacy_intel/` 内的反编译字符串、DLL 导出、历史 mdb 样本为准，不再依赖现场发卡器实测。

### 9.1 老系统卡台账规则

老系统 `CardInfo` 真实 SQL 路径：

```sql
INSERT INTO CardInfo VALUES(:ID,:CardData,:Holder,:Operator,:BeiZhu,:Status)
UPDATE CardInfo SET Status=:a WHERE CardData=:a1
DELETE FROM CardInfo WHERE ID=:a
SELECT * FROM CardInfo WHERE CardData=:a
SELECT * FROM CardInfo WHERE Mid(CardData,19,8)=:a
```

关键状态：

| 表/字段 | 规则 |
|---|---|
| `CardInfo.Status` | 中文字面值：`客人卡`、`已注销`、`已挂失`、`已过期`，以及 `授权卡`、`记录卡`、`房号设置卡`、`时间设置卡`、`挂失卡`、`组号设置卡`、`退房卡`、`组控卡`、`应急卡`、`总控卡`、`楼栋卡`、`层控卡` |
| `GuestInfo.Flag` | 英文 token：`WalkIn`、`TeamIn`、`Copy`、`CheckOut`、`Reservation`、`CancelReservation` |
| `RoomInfo.Status` | `VC/OH/OT/TO/VD/OO/OC_WalkIn/OC_Team`，中文展示为空净房、钟点房、预订房、催租房、脏房、维修房、散客房、团体房 |

前台卡动作：

| 老系统按钮 | DLL | 物理卡 | 数据库 |
|---|---|---|---|
| `IssueCard 发卡` | `GuestCard` | 写客人卡 | `CardInfo.Status=客人卡`，`GuestInfo.Flag=WalkIn`，`RoomInfo.CardCount+1` |
| `CardErase 卡片注销` | `CardErase` | 擦回空白卡 | 只把 `CardInfo.Status` 改为 `已注销`，不等于退房 |
| `CheckOutByCard 有卡退房` | `CardErase` | 擦回空白卡 | 卡注销 + 客人 `CheckOut` + 房态变更 |
| `CheckOutNoCard 无卡退房` | 不调 DLL | 不动卡 | 只改客人 `CheckOut` 和房态 |
| `TeamCheckOut 团体退房` | 不调 DLL | 不动卡 | 批量退房；老系统提示稍后逐张注销房卡 |
| `Postponed 续房` | `GuestCard` | 重写有效期 | `GuestInfo.Flag=Copy` |
| `Change 换房` | `GuestCard` | 写新房号客人卡 | 换房号，两个房间卡数调整 |
| `BlackNameCard 挂失` | `LimitCard` | 写挂失卡 | 目标卡标记 `已挂失` |

员工/工程卡共 13 类：`IniCard` 授权卡、`RecordCard` 记录卡、`RoomSetCard` 房号设置卡、`TimeSetCard` 时钟设置卡、`LimitCard` 挂失卡、`GroupSetCard` 组号设置卡、`GuestCard` 客人卡、`CheckOutCard` 退房卡、`GroupCard` 组控卡、`EmergencyCard` 应急卡、`MasterCard` 总卡、`BuildingCard` 楼栋卡、`FloorCard` 层控卡。

注意：`CheckOutCard 退房卡` 是员工查房刷锁用的功能卡，不是前台“卡片注销”。前台注销必须是 `CardErase`。
