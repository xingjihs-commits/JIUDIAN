## 0. 一句话现状

**ProUSB V9 HL-M1 发卡器不是全死，`initializeUSB(d12=1)` 可成功打开 USB 通道（ret=0）**。CardLock.exe 启动时还会调用 `initializeUSB(d12=0)`，该阶段曾返回 259（锁定），所以不要让 CardLock 直接走完整初始化。

**已验证活路**：32 位 PowerShell/ctypes 跳过 `d12=0`，只用 `d12=1 + ReadCard`，已成功读出授权卡：`C92B20B7DF48809C00021483E51483E5`。

## 3. ✅ 已确认事实（v2.0 + v2.1 总和）

### 3.1 硬件
- 发卡器型号：ProUSB V9 HL-M1，VID_9915 PID_1116
- 锁定状态：firmware 层，OS / USB 复位无效
- 半死特征：`d12=1` 能开句柄并读卡；`d12=0` 曾触发 259 锁定

### 3.2 V9RFL.dll 函数表（reports/v9_calling_conventions_20260531_0445.txt）
- 77 个导出，CardLock 只用 8 个
- 关键函数（可能需要的）：
  - `GetDLLVersion` (0x1420) — 只读
  - `initializeUSB` (0x1a50) — 1 参 → ret 0=成功；ret 1/259=失败/锁定
  - `ReadCard` (0x1be0) — 1 参（buf 指针），读卡到 buf
  - `Buzzer` (0x29f0) — 2 参，纯硬件信号
  - `proTest` (0x1600) — 0 参，自测
  - `CloseUSB` (0x1ab0) — 0 参，关句柄

## 4. 🎯 真正的下一步（按 ROI 排序，v2.1 重写）

### 🥇 第 1 优先：最小读卡探针（5 分钟，零写卡风险）

**核心思路**：跳过 `initializeUSB(0)`，只用 `initializeUSB(1)` + `ReadCard`。

**步骤**：
1. 物理：发卡器表面**放授权卡**（这次需要它在场）
2. 32-bit Python 加载 `V9RFL.dll`
3. 调 `initializeUSB(1)` —— 期待 ret=0
4. 调 `ReadCard(buf, 36)` —— 期待返回授权卡 16 字节 payload
5. 验证 payload 前 8 字节是 `C92B20B7DF48809C`
6. 调 `CloseUSB()` —— 优雅关闭
7. 实测已读到 #13：`C92B20B7DF48809C00021483E51483E5`，无需追加重复记录

**失败容忍**：
- 如果 `initializeUSB(1)` 也开始返 1/259 → 发卡器又进入锁态，跳第 2 优先或先 CloseUSB 后重试
- 如果 `ReadCard` 失败 → 试 `ReadRecord` / `DirectReadUSB` —— 都是只读
- **无论何时不调任何 W*/Card*/Set*/Erase 函数**

**风险**：极低。只读路径，授权卡放发卡器上 5 秒就能拿走。
