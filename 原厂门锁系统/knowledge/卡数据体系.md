# 卡数据体系 — 全盘推算结论

> **本文件基于 30+ 张真实样本卡的统计推算，记录所有卡型的 payload 结构、校验和、时间戳编码公式。**
> 生成时间：2026-06-05 | 工具：`tools/dev/_crack_analysis.py`

---

## 一、总览：Payload 通用结构

所有卡型的 16 字节 payload 共用以下骨架：

```
字节偏移  长度    字段       说明
0~3       4      Magic      固定 C92B20B7
4~5       2      Site       酒店码（每家酒店不同）
6~7       2      LockNo     锁号/通配
8         1      Salt       盐值（通常 00）
9         1      Type       高4bits=卡类型, 低4bits=序号
10~13     4      Body       类型相关数据
14~15     2      Checksum   校验和
```

## 二、MasterCard（总卡）

### 样本统计

- **FF00 组**：6 张，dlsCoID≈2826423（酒店1）
- **AA00 组**：12 张，dlsCoID≈170（酒店2）
- **相同酒店码在所有组内一致**

### 结构

```
偏移    值        说明
0~3     C92B20B7  Magic 固定
4~5     FF00      Site码 → 对应 dlsCoID=2826423
        AA00      Site码 → 对应 dlsCoID=170
6~7     6C61      LockNo（每张卡不同，可能是天+序号）
8       00        盐值固定
9       B9~BE     高半字节 B = MasterCard
                 低半字节 = 序号（B9, BA, BB, BC... 不代表严格递增）
10~11   xxxx      Body 高2字节（与天+序号相关）
12~13   xxxx      Body 低2字节（与时间相关）
14      xx        校验和第一字节
15      FB        校验和第二字节 **始终固定 = 0xFB**
```

### 校验和

```
byte[15] = 0xFB 恒成立（FF00 6张 + AA00 12张 + 楼栋卡1张 + 楼层卡1张）

byte[14] = ?  每张卡不同
```

**系统卡（MasterCard / BuildingCard / FloorCard）全部以 0xFB 结尾。**
这强烈暗示 0xFB 是系统卡类型的"签名尾字节"，与 GuestCard/AuthCard 区分。

→ 生成系统卡时，直接设 byte[15] = 0xFB

## 三、BuildingCard（楼栋卡）& FloorCard（楼层卡）

### 楼栋卡（仅1张样本）

```
C92B20B7FF00B30100C115DAD925DDFB
    Site=FF00  LockNo=B301  Type=C1  byte[14]=DD  byte[15]=FB
    类型: C = BuildingCard, 序号 = 1
```

### 楼层卡（仅1张样本）

```
C92B20B7FF00030100D515DAE016D5FB
    Site=FF00  LockNo=0301  Type=D5  byte[14]=D5  byte[15]=FB
    类型: D = FloorCard, 序号 = 5
```

### 推断模板

```
[0:4]  C92B20B7        Magic
[4:6]  FF00            Site（按酒店变）
[6:8]  LockNo          （楼栋卡=B3xx, 楼层卡=楼层号）
[8]    00              Salt
[9]    Cx / Dx         类型字节
[10:14] Body           时间戳/数据
[14:15] xx / FB        校验和, byte[15]=FB
```

## 四、AuthCard（授权卡）

### 样本统计

14 张卡（含1张重复），全部来自同一家酒店（byte[4:6] = DF48）。

### 结构 — 重大发现

```
偏移    值        说明
0~3     C92B20B7  Magic 固定
4~5     DF48      Site码（固定，这家酒店特有）
6~7     809C      LockNo 固定不变！
8       00        盐值
9       0x        高半字节=0（AuthCard）
                 低半字节=序号（0,1,2,3,5,6,7,8,14,15 观测到）
10~12   xxxxxx    3 字节 Token
13~15   xxxxxx    3 字节 Token **完全重复**
```

### Token 分析

最后 6 字节 = 3 字节 Token 写入 2 次。

Token = f(Site, Seq, Timestamp)

无法从静态样本推导算法 → 需要 algo_study 矩阵现场采集。

**短期方案**：建立 {seq: token} 查找表，用已知样本的 token。

### 授权卡 vs 总卡的 Site 码为何不同？

| 卡型 | Site 码 |
|:-----|:--------|
| AuthCard | DF48 |
| MasterCard | FF00 / AA00 |

同一酒店（DF48 对应酒店）的 MasterCard 应该用 FF00。

→ Site 码不是简单的 HotelID 哈希，可能是 dlsCoID 的不同编码。

## 五、GuestCard（客人卡）

### 校验和 — 当前代码错误

```python
# 文件: _crypto_complete.py guestcard_bcd_transform() 第210行
# 当前:
checksum = sum(payload[:14]) & 0xFF

# 样本 C92B20B781838E000061160800161300:
#   sum[:14] = C9+2B+20+B7+81+83+8E+00+00+61+16+08+00+16 = 0x363
#   sum[:14] & 0xFF = 0x63
#   实际 byte[14] = 0x13
#   不匹配！
```

正确算法未知，需更多样本。

### 时间戳

```
byte[12] = yy (BCD year)
byte[13] = mmmmdddd (BCD month + day)
byte[14:15] = 校验和 + 00
```

→ GuestCard 时间戳只有年月日，没有时分秒。

## 六、生产代码修复清单

### prousb_v9.py `_issue_legacy_system_payload()`

**问题**：
1. 生成的 payload **不含 C92B20B7 Magic 头**
2. **没有校验和**

**修复方案**：

```python
def _issue_legacy_system_payload(dls_co_id, card_type, extra_hex=""):
    """
    生成系统卡 payload（C92B20B7 格式）
    """
    # 1. Magic 头
    payload = bytearray(16)
    payload[0:4] = bytes.fromhex("C92B20B7")
    
    # 2. Site 码
    site_code = dls_co_id & 0xFFFF
    payload[4] = (site_code >> 8) & 0xFF
    payload[5] = site_code & 0xFF
    
    # 3. LockNo
    payload[6:8] = bytes.fromhex(extra_hex[:4].ljust(4, "0"))
    
    # 4. Salt
    payload[8] = 0x00
    
    # 5. Type
    type_map = {
        "MasterCard": 0xB0,
        "BuildingCard": 0xC0,
        "FloorCard": 0xD0,
        "EmergencyCard": 0xA0,
    }
    base = type_map.get(card_type, 0xB0)
    payload[9] = base | (seq_num & 0x0F)
    
    # 6. Body data + timestamp
    ... # 按原逻辑填充 byte[10:14]
    
    # 7. Checksum (系统卡 byte[15] = 0xFB)
    payload[15] = 0xFB
    # byte[14] 的算法待定 — 先用 0x00 占位
    payload[14] = 0x00
    
    return payload.hex().upper()
```
