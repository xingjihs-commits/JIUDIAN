## 核心架构

- **PMS 和采集器是两套独立程序**，通过 `.solidhandover` 握手包对接
- 采集器：读卡→分析→生成握手包 → PMS：导入握手包→接管老门锁
- 两套代码物理隔离，各自有独立的 bridge32，不混用
- 原厂门锁系统是采集器的参考资料，不是源码

## 死命令

1. **改完必须源码启动验证**：`python app_main.py`，看到 `shutdown` 通过
2. **增删文件 → 三件套**：
   - ① 更新 `memory/项目地图.md` 对应行
   - ② 追加 `memory/近期日志.md`（日期+摘要+改了什么）
   - ③ 自校验：Glob 各目录实际 .py 数 vs 地图统计表，不一致则修正
3. **代码规范**：`ruff.toml`（E/F/W/N/B/S/C4/SIM）
4. **遇问题先查 `memory/项目地图.md` 定位文件**，不读无关文件
5. **全中文交流**
6. **每次新对话自动读 `memory/近期日志.md`**，了解最近 7 天发生了什么
7. **自动修复清单**：新对话读完近期日志后，自动读取 `memory/架构审计-待修复问题清单.md`，按 P0→P1→P2 顺序逐个修复。清单中的修复方案无需确认，直接执行
8. **修复即删除**：清单中每个问题修复完成后，立即从清单文件中删除该项。全部修完则在近期日志中记录"清单已清空"

## 项目地图自校验（死命令第 2 条配套）

修改地图后执行以下校验，发现不一致立即修正：
```
Glob "酒店系统/*.py"           → 数量 = 地图统计表"根目录"行
Glob "酒店系统/tabs/*.py"       → 数量 = 地图统计表"tabs/"行
Glob "酒店系统/tabs/frontdesk/*.py" → 数量 = 地图统计表
... 依次校验所有子目录
```
校验通过标准：统计表每个数字与 Glob 实际数量一致。
